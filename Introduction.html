The Problem

    In order to provide a reliable and sustainable M2M/IoT application system it is very important to identify any misbehaviors or anomalies. Such anomalies can be caused by malfunction of sensors, malicious attacks, intrusion of networks, or unusual/interesting events in the remote monitoring systems.

    Goal

    For detecting uncertain anomaly types, we provide diverse anomaly detection algorithms in this toolbox. Each researcher can use different algorithms to find types of anomalies in their own dataset. Aim at uncertain anomaly types, manual data science effort prohibits wide adoption of anomaly detection for IoT. Our next step is to provide (semi)-auto algorithm selection.

    Ingredients

    This toolbox includes 3 principle components shown as following:

    1. Algorithms of different types
      • Angle-based, distance-based. distribution-based, and others
      • Organized in toolbox
    2. Benchmark: Representative Dataset
      • Anomaly type: point, contextual, collective
      • Input type: single/multi-sensors, single/multi-readings
    3. Criticisms
      • Performance of each (A,B)
