• Anomaly detection is important for large-scale data where no or very few label information available
  • An anomaly detection toolbox servers an all-in-one method for general purpose given IoT data owning different characteristics
  • Proposed toolbox = Algorithms + Benchmarks + Criticism/Assessment
  • Algorithms: angle based, distance based, distribution based, etc.
  • Benchmarks:
    • datasets with point anomalies, contextual anomalies, or collective anomalies
    • data from single sensor, multiple readings; multiple sensors, single reading; and multiple sensors and multiple readings
  • Criticisms include past performance for each (method, dataset) pair
